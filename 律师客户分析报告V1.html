三

客户需求提炼

按频率和价值排序 + 深层逻辑分析
优先级 需求场景 占日常比例 痛苦程度 AI可行性
⭐⭐⭐ 微信/邮件法律咨询回复 ~75%（微信）+ 10-15%（邮件） 高（复杂咨询半天） 高（已有验证）
⭐⭐⭐ 合同起草（无中生有） 中高频 中（有模板但耗时） 高（已在用AI）
⭐⭐ 合同审核+输出修订痕迹 中频 高（细节多，标准个人化） 中（需接入个人数据库）
⭐⭐ 客户需求澄清（挖真实意图） 高频（每单都有） 中（靠经验，难标准化） 中（需要经验萃取）
⭐⭐ 法律法规快速检索 高频 中（有工具但需整合） 高
⭐ 案件跟进通知回复 中频 低（内容简单） 高（低价值，易实现）
核心需求的深层逻辑：表面看是「AI 帮我写」，深层是「AI 要懂我、懂我的客户」——通用工具做不到的正是定制化价值所在。

📱 需求1：微信回复助理

原话「我预判他会用AI审核我的内容，所以我要写得比AI更专业更详细」
深层「这个客户是谁」决定「该说多少、多详细」
本质不是生成文字，而是根据客户画像决定输出深度

📝 需求2：合同起草

原话「客户说走账，但真实意图是节税——直接给AI他会搞出走账内容」
深层客户说的 ≠ 客户要的，需要意图翻译
本质不是执行指令，而是先翻译意图，再生成合同

🔍 需求3：合同审核

原话「阿尔法是60分市场水平，我要的是符合我自己标准的80分」
深层标准是个人化的，不存在通用80分
本质不是通用审核，而是学习我过往改合同的偏好和注意点

⚡ 需求4：多线程处理

原话「我们每天都是同步处理很多工作，多线程的才会买」
深层时间是律师唯一资产，多任务并发是真实需求
本质不是单点替代，而是让AI并发处理10件事，每件都及格
四

AI Agent Skill 赋能点

6 个 Skill，覆盖沟通→起草→审核→检索→画像→法规全链条
设计原则：每个 Skill 的核心差异点都是「了解律师 + 了解客户」，而不是做一个通用的文字生成工具。

Skill 1 · 律师微信回复起草器

场景客户发来微信咨询 → 律师10秒生成专业回复草稿
输入客户原文 + 客户标签（法律知识/AI使用/大小客户）+ 律师偏好
输出适配场景深度的专业回复草稿 + 关键法条引用 + 简短/详细备选
✦ 知道「这个客户」要简短还是详细 · 知道律师自己的表述风格 · 预判客户会用AI反查，所以输出更严谨

Skill 2 · 客户需求翻译器

场景客户说了一句话 → AI帮律师问出真实需求
输入客户的模糊请求（「你有没有营销服务合同模板」）
输出3个澄清问题 + 初步意图判断 + 合同风格建议
✦ 提炼经验律师的追问逻辑 · 防止「直接执行」导致方向跑偏 · 将口语意图翻译成精确指令

Skill 3 · 合同起草器（意图驱动）

场景律师确认客户意图后 → AI生成完整合同初稿
输入合同类型 + 代理方立场 + 客户真实意图 + 特殊关注点
输出完整合同初稿 + 重点条款说明 + 客户立场利益清单
✦ 基于意图起草，非模板复制 · 接入律所历史合同数据库 · 远超「豆包给建议」的执行深度

Skill 4 · 合同审核器（带修订痕迹）

场景客户发来对方合同 → AI输出带修订痕迹的专业版本
输入原始合同 + 代理立场 + 审核重点 + 历史审核标准数据库
输出带修订痕迹的Word（可直接交付）+ 修订理由 + 未覆盖点提示
✦ 直接输出修订痕迹版本（超越阿尔法）· 接入个人库后60→80分 · 明确标注「需人工复核」的部分

Skill 5 · 客户画像积累器

场景基于历史沟通记录，建立每个客户的沟通特征档案
输入与客户的历史微信/邮件记录 + 客户身份标签
输出客户沟通偏好报告 + 高频疑虑点 + 追问触发词 + 定制话术模板
✦ 不用靠律师记忆「小林子这类客户」· 每次回复前自动调取档案 · 持续学习，越用越懂

Skill 6 · 法规快查+意见匹配器

场景遇到无明确法规的边缘问题 → AI完成4步检索流程
输入法律问题描述 + 当前客户情况背景
输出相关法条 + 专家意见汇总 + 适用性判断 + 严谨表述草稿
✦ 不只检索，还帮你判断「适不适用」· 输出即可用的严谨表述，非原始资料堆砌 · 半天→20-30分钟
